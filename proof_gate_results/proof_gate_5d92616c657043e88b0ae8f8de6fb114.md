**Gate logic grounding from the repository:**

The `recon_bravo_then_corecritical` fallback is reached only when every specific exploit-family branch in `scoring.py` is skipped. [1](#0-0) 

The `entity_type` stays `unknown_protocol` (not `unverified_contract`) because `_entity_type()` only promotes to `unverified_contract` when `verified is False`; a `None` return from `_is_verified()` does not satisfy that condition. [2](#0-1) 

`_is_verified()` returns `None` (not `False`) when the source payload is empty — the pipeline never received a source response, it did not confirm the contract is unverified. [3](#0-2) 

The empty `deployer_address` is a direct consequence of discovery via `erc20_transfer_log`: the `Deployment` dataclass is constructed with `deployer=""` in that path. [4](#0-3)

    def _entity_type(self, tags: list[str], verified: bool | None) -> str:

def _is_verified(source: dict[str, Any]) -> bool | None:
    if not source:
        return None
    if "SourceCode" in source:
        return bool(str(source.get("SourceCode") or "").strip())
    return None
