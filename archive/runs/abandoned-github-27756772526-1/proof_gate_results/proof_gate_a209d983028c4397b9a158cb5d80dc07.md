**Why REJECT and not NEEDS_LOCAL_PROOF:**

The proof-gate schema [1](#0-0)  only permits promotion to `NEEDS_LOCAL_PROOF` when there is an exact code path or selector to test against. The hard stop in the hunt plan [2](#0-1)  requires a reproducible path before any escalation. The evidence requirements [3](#0-2)  explicitly demand exact entrypoints that move funds — none exist here because `verified_source` is null and no selector extraction was submitted with this gate. The rejection is a gate-process rejection (resubmit after recon), not a finding rejection.
